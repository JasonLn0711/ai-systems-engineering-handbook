# Reference Answer — Onboarding Pack

> Instructor / TA only.

## Evidence Index Example

| Day | Evidence | What it proves |
|---|---|---|
| 1 | AI Gateway architecture | Request lifecycle and control plane |
| 2 | Agent governance | Registry, policy, memory, review, audit |
| 3 | Red-team guardrails | Testable PII/security controls |
| 4 | RAG/tool gateway | Governed data/tool access |
| 5 | K8s/GPU serving | Deployability and sizing |
| 6 | Demo/memo | Integrated, scoped, measurable evidence |

## First-Week Questions Example

```text
What model-serving stack is currently used?
Where are audit logs stored?
What data cannot leave the customer environment?
Which tool actions create side effects?
What evaluation set already exists?
What milestone acceptance criteria are agreed?
```

## First-30-Days Plan Example

| Week | Goal | Evidence |
|---|---|---|
| 1 | Map current stack and workflows | repo map, model inventory, milestone notes |
| 2 | Add minimum governance | request ID, audit schema, PII gate |
| 3 | Add evaluation/red-team | RAG eval, ASR eval, injection tests |
| 4 | Package delivery | K8s notes, GPU sizing, acceptance checklist |

## Common Mistakes

- Evidence index does not state what artifacts prove.
- First-week questions are too generic.
- 30-day plan starts with a rewrite instead of mapping.
- Mock answers do not include metrics or next validation.
