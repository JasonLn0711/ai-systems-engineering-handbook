# Instructor Guide — Day 7: Onboarding Pack

## 1. Teaching Goal

Students learn to convert sprint work into an onboarding and review packet.

## 2. Pre-Class Diagnostic

1. Which artifact best proves architecture reasoning?
2. Which artifact best proves security readiness?
3. Which artifact best proves deployment readiness?
4. What first-week question would reveal the biggest hidden risk?
5. What should be validated after the 7-day sprint?

## 3. 150-Minute Flow

| Time | Activity | Output |
|---:|---|---|
| 0-10 | Diagnostic | Missing evidence list |
| 10-35 | Onboarding pack map | File map |
| 35-60 | Evidence index | Day 1-6 proof table |
| 60-85 | First-week questions | Question bank |
| 85-110 | First-30-days plan | Weekly plan |
| 110-135 | Mock review checklist | Q&A scaffold |
| 135-150 | Next validation path | Acceptance criteria |

## 4. Failure Gallery

### Failure 1: Pack is just links

Fix by adding what each artifact proves and what validation remains.

### Failure 2: First-30-days plan starts by rewriting

Fix by making Week 1 a system mapping week.

### Failure 3: Mock answers are defensive

Fix by answering with architecture, metric, control, limitation, and next
validation.

### Failure 4: 30-day bootcamp replaces 7-day completion

Fix by treating 30 days as next spiral, not a blocker for the 7-day evidence
packet.

## 5. TA Workflow

1. Check evidence index.
2. Inspect first-week questions.
3. Grade first-30-days plan.
4. Review mock checklist.
5. Verify source boundary.
