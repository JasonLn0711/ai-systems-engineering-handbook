# Enterprise AI Architecture Sprint — Day 7: Onboarding Pack

Day 7 turns the sprint artifacts into an onboarding packet and first-30-days
plan. The learner should leave with a public-safe evidence index, first-week
fact-finding questions, 30-day plan, mock review checklist, and next validation
path.

## Day Metadata

```text
Day number: Day 7
Topic: Onboarding Pack + First 30 Days Plan
Sprint artifact: master-knowledge-base/08-first-30-days-learning-checklist.md + 30-day-spiral-bootcamp.zh-TW.md
Owning module: modules/10-enterprise-delivery-fae/
Supporting lab: labs/local-dev/
Target learner: sophomore CS students / junior engineers
Prerequisites: Day 1-6 evidence artifacts
Expected student deliverables: onboarding pack, first-week questions, first-30-days plan, portfolio/evidence index, mock review checklist
Next gate: day-08-lab-handoff.md
```

## Learning Objectives

1. Build an evidence index that links the 7-day artifacts into one coherent
   enterprise AI architecture story.
2. Write first-week fact-finding questions for stack, model serving, data,
   deployment, security, evaluation, logs, and milestones.
3. Convert the 7-day sprint into a first-30-days plan.
4. Prepare mock technical review answers that cite architecture, metrics,
   controls, limitations, and next validation.
5. Define the next validation path without turning the 7-day sprint into the
   30-day bootcamp.

## Source Boundary

Use only public-safe, generalized system patterns and synthetic evidence. Do not
include private interview text, customer secrets, credentials, real customer
data, personal contact routes, salary/offer details, or unpublished company
claims.
