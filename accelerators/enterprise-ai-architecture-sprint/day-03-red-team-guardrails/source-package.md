# Source Package — Day 3: Red-Team Guardrails

This file records source boundaries and reference routes for Day 3. It is for
instructors, TAs, and repo maintainers, not a default student handout.

## Source Boundary

Allowed:

- generalized AI security patterns;
- public-safe synthetic scenarios;
- public standards and documentation;
- synthetic Campus IT, sales coach, manufacturing support, banking knowledge
  assistant, or healthcare staff-review intake support examples.

Not allowed:

- private transcripts;
- customer secrets;
- credentials;
- personal contact routes;
- salary or offer details;
- unpublished company claims;
- identifiable personal data;
- real helpdesk tickets;
- real internal SOP content.

## Technical Language

Day 3 uses public-safe language for:

- prompt injection;
- PII detection and minimization;
- guardrail gates;
- tool abuse;
- privilege escalation;
- memory poisoning;
- retrieval manipulation;
- audit evasion;
- human review bypass;
- remediation backlog.

Tool names such as Presidio, OWASP, NIST, or Microsoft AI Red Team may be used
as reference routes for instructors, but the student deliverable is the
artifact contract: taxonomy, event schema, cases, report, and backlog.
