# Core Mental Models

## AI Model vs AI Application vs AI System

- AI model: performs inference or generation.
- AI application: wraps the model in product logic.
- AI system: integrates model, workflow, data, infrastructure, permissions, monitoring, security, and evaluation.

## Deployment Is A Constraint System

Public cloud, private cloud, on-prem, and hybrid deployment are not just hosting choices. They define data boundaries, latency, cost, operational ownership, incident response, and compliance review.

## RAG Is A Data Pipeline

RAG quality depends less on a single prompt and more on ingestion, parsing, metadata, indexing, retrieval, reranking, context construction, and evaluation.

## Gateway Is Governance Infrastructure

An AI Gateway is not only a routing layer. It is where model access, tool access, knowledge access, permission boundaries, policy checks, memory controls, approval gates, and audit logs become system behavior.

## Voice AI Is A Latency Pipeline

Voice systems combine audio capture, VAD, ASR, LLM reasoning, TTS, playback, interruption, and state management. A good design budgets latency by stage.

## Codex Needs Architecture

AI-assisted coding works best when spec, SDD, module boundaries, data contracts, and tests are written before implementation.
