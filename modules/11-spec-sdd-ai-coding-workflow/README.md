# Module 11: Spec, SDD, And AI-Assisted Coding Workflow

## Purpose

Teach how to use Codex and AI agents without losing architecture.

## Core Questions

- Why should spec and SDD come before implementation?
- How do module boundaries, interface contracts, data flow, error handling, and tests constrain AI-generated code?
- How does vibe coding fail, and how can review gates reduce those failures?

## Suggested Chapters

- Why Spec Before Coding.
- How To Write Product Spec.
- How To Write SDD.
- Module Boundary And Interface Contract.
- Design Pattern For AI-Generated Code.
- How Vibe Coding Fails.
- Context Hygiene For Codex.
- Reviewing AI-Generated Code.
- Testing And Regression.
- From Spec To Implementation Plan.

## First Labs

- Write a spec for a RAG system.
- Write an SDD for an AI Gateway.
- Refactor over-coupled AI-generated code.
