# Module 01: Linux, OS, And Networking

## Purpose

Build the operating-system and networking foundation required for local and enterprise AI deployment.

## Core Questions

- Why do AI systems engineers need Linux?
- How do processes, services, daemons, systemd, ports, sockets, DNS, TLS, SSH, firewalls, users, permissions, and logs affect deployment?
- How do you debug an AI service that works locally but fails in an enterprise environment?

## Suggested Chapters

- Linux Mental Model.
- Process, Service, Daemon, And systemd.
- Filesystem, Permission, User, And Group.
- Port, Socket, DNS, And TLS.
- SSH, Firewall, And Network Debugging.
- Logs And Resource Monitoring.
- Common On-Prem Debugging Patterns.

## First Labs

- Run a simple AI service under systemd.
- Debug a port, firewall, or DNS failure.
