# Module 09: Security, Red Teaming, And AI Risk

## Purpose

Teach AI security as system security: data, permissions, tools, model behavior, auditability, and recovery.

## Core Questions

- How do IAM, zero trust, audit logs, PII, guardrails, prompt injection, and tool boundaries apply to AI systems?
- How can RAG, tools, and agents leak or misuse data?
- What should an AI red-team report test?

## Suggested Chapters

- AI Security Is System Security.
- IAM, Zero Trust, And Audit Log.
- OWASP, NIST, And Enterprise Baseline.
- PII And Data Protection.
- Guardrails, Input, Output, And Tool Boundaries.
- Prompt Injection.
- Data Exfiltration Through RAG.
- Malicious Documents And KB Poisoning.
- User-Side Red Teaming.
- Agent Red Teaming.
- Red-Team Report Template.
- Security Acceptance Test.

## First Labs

- Build an audit log.
- Test prompt injection.
- Test PII leakage.
- Simulate agent tool abuse.
