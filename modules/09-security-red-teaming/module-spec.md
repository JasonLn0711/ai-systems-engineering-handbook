# Module Spec: Security, Red Teaming, And AI Risk

## Purpose

Teach AI security as system security: IAM, data boundary, PII, prompt injection, tool abuse, RAG exfiltration, memory leakage, audit logs, and red teaming.

## Learning Outcomes

- Explain how traditional security and AI-specific risks combine.
- Design acceptance tests for AI security and governance.
- Identify prompt injection, malicious documents, and cross-agent privilege risks.

## Required First Chapters

- `01-ai-security-is-system-security.md`
- `05-guardrails-input-output-and-tool-boundaries.md`
- `10-agent-red-teaming.md`
