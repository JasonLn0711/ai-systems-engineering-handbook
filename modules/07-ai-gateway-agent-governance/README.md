# Module 07: AI Gateway And Agent Governance

## Purpose

Teach scalable control of multiple agents, tools, knowledge bases, permissions, memory, and audit logs.

## Core Questions

- Why does enterprise AI need an AI Gateway?
- How do MCP servers, skills, adapters, tools, knowledge bases, memory, and policy engines fit together?
- How can one shared SQL or knowledge source be exposed safely to project-specific agents?
- How do we prevent tool abuse, memory leakage, and cross-agent privilege escalation?

## Suggested Chapters

- Why AI Gateway Exists.
- Agent, Tool, Memory, And Knowledge Boundary.
- MCP Server Mental Model.
- Skill And Adapter Architecture.
- Policy Engine And Approval Gates.
- Agent Registry And Tool Registry.
- Memory Governance And Permission Boundary.
- Cross-Agent Privilege Escalation.
- Audit Log And Decision Trace.
- General vs Task-Specific Governance.
- AI-Generated Spec-To-Skill Pipeline.
- Governance Evaluation.

## First Labs

- Build a simple AI Gateway mock.
- Register two agents over one knowledge base.
- Add a policy gate and audit log.
- Simulate cross-agent data leakage.
