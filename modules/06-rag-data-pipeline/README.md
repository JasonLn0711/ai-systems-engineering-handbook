# Module 06: RAG And Data Pipeline

## Purpose

Teach enterprise knowledge ingestion, retrieval, metadata, reranking, evaluation, and data contracts.

## Core Questions

- Why is RAG a data pipeline rather than a prompt trick?
- How should documents become chunks, metadata, embeddings, search results, reranked contexts, and cited answers?
- How do SQL and knowledge-base sources work together?

## Suggested Chapters

- RAG Mental Model.
- Document Ingestion And Parsing.
- OCR, PDF, HTML, Table, And Image.
- Chunking Strategies.
- Metadata Generation.
- Embedding And Vector Search.
- Reranker And Hybrid Search.
- Query Rewriting And Intent Classification.
- Context Construction.
- RAG Evaluation.
- RAG Failure Modes.
- RAG For Enterprise KB And SQL.

## First Labs

- Build a basic RAG pipeline.
- Add metadata filtering.
- Add a reranker.
- Test hallucination and missing context.
