# Module 05: LLM Application Architecture

## Purpose

Teach how prompt-based applications become maintainable AI services.

## Core Questions

- What changes when a prompt becomes a backend service?
- How do routers, structured output, tool calling, retries, timeouts, fallback, streaming, logs, and evals make an LLM app operable?
- Where should boundaries exist between prompt, code, data, tool, and policy?

## Suggested Chapters

- From Prompt To AI Service.
- API Service, Router, And Gateway.
- Prompt Template And Structured Output.
- Tool Calling And Function Boundary.
- Timeout, Retry, And Fallback.
- Streaming Response.
- Observability For LLM Apps.
- Evaluation And Regression Test.

## First Labs

- Build a FastAPI LLM service.
- Add structured output parsing.
- Add evaluation tests.
