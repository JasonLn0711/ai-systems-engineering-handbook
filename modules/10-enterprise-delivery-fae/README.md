# Module 10: Enterprise Delivery And FAE Workflow

## Purpose

Teach customer-facing delivery as a system-design discipline.

## Core Questions

- How do customer workflow, machine room constraints, network limits, data policy, and acceptance criteria become engineering requirements?
- What should be checked before onsite deployment?
- How should incidents and handoffs be documented?

## Suggested Chapters

- Enterprise AI Delivery Lifecycle.
- Requirement Interview And Site Survey.
- Machine Room And Network Checklist.
- Customer Data And Security Boundary.
- Deployment Plan And Rollback Plan.
- Acceptance Criteria For Generative AI.
- Onsite Debugging Method.
- Customer Communication For Technical Risk.
- Handover And Maintenance.

## First Labs

- Write a site survey.
- Write a deployment plan.
- Debug a fictional customer incident.
